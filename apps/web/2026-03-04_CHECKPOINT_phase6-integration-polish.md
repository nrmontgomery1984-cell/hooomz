## Blockers Closed

### Blocker 1: Selection → Quote Integration

Bridge service that syncs confirmed material selections into quote line items.

**Schema changes:**
- Added `LineItemSourceSchema` enum (`manual`, `material_selection`, `labour_estimation`) and optional `source`/`source_id` fields to `LineItemBaseSchema` in `packages/shared-contracts/src/schemas/index.ts`
- IDB v30 cursor migration already sets `source='manual'` + `source_id=null` on existing records — no new migration needed

**Service:**
- `materialSelectionToLineItems.service.ts` — `generateLineItemsFromSelections(projectId)` reads confirmed selections, upserts line items with `source='material_selection'`, logs `material.synced_to_quote` activity event
- `countExistingMaterialLineItems(projectId)` for button label toggle
- Registered on `Services` interface + `initializeServices()`

**Hooks:**
- `useSyncSelectionsToQuote()` — mutation calling the service
- `useMaterialLineItemCount(projectId)` — query for existing count

**UI:**
- `RoomMaterialsSummary.tsx` — "Push selections to quote" / "Re-sync selections to quote" teal button, visible after confirmed selections exist
- Quote page (`sales/quotes/[id]/page.tsx`) — line items grouped by source (Materials / Labour / Other) with subtotals when multiple sources present
- `QuotePDF.tsx` — same source-grouped rendering in PDF output

**Files created:** `materialSelectionToLineItems.service.ts`
**Files modified:** `shared-contracts/schemas/index.ts`, `services/index.ts`, `useMaterialSelections.ts`, `RoomMaterialsSummary.tsx`, `quotes/[id]/page.tsx`, `QuotePDF.tsx`
